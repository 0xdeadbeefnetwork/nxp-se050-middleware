2.4. Release v04.07.00¶

2.4.1. External module update¶

  • Mbed TLS 3 is updated to version 3.6.2.

  • Paho mqtt module in external folder updated to version 1.3.14.

  • Protobuf (to auto-generate file for nxp_iot_agent/src/protobuf/) version is updated to nanopb-0.3.9.8. (https://github.com/nanopb/nanopb/blob/nanopb-0.3.9.8/CHANGELOG.txt).

  • MCU SDK updated to version 2.16

    • Mbed TLS 2.x is updated to 2.28.5

    • FreeRTOS Kernel updated to version 11.0.1

    • FreeRTOS-plus-TCP updated to version 4.0.0

    • AWS-Iot is updated to use Mbed TLS PSA crypto

2.4.2. New platform support¶

  • MCXN-947 support added (Supports Mbed TLS 3.x version only).

  • MCXA-153 support added (No host crypto support available for MCXA-153).

2.4.3. SSSCLI Changes¶

  • Cryptography module updated to 44.0.0.

    • Limitation: The following curves are not supported anymore - brainpoolP160r1, brainpoolP192r1, brainpoolP224r1, brainpoolP320r1, secp160k1, secp192k1, secp224k1.

  • setuptools module updated to 75.3.0.

  • Get memory command to see the available RAM and Flash memory of secure element is added. Refer - List of ssscli commands

2.4.4. Functional Changes / New feature support¶

  • Platform SCP03 support extended for AES-256 (32 byte) keys. SE05X Rotate PlatformSCP Keys Demo

  • seTool updated for OpenSSL 3.x.y host crypto. (Tested with version - OpenSSL 3.0.14)

  • Signal handlers are added to handle signals SIGTERM and SIGINT in access manager and client for clean exit.

  • Access manager bug fix - corrected the check for APDU limit error (0x66A6).

  • Pre-built vcom and get info binaries added for MCXA-153 and MCX-947 boards

  • T=1oI2C Stack updated to remove the endianness dependent code.

  • smComSocket_Transceive functions are updated to retry on EINTR error also.

  • Fixes for memory leaks.

    • emsa_decode_and_compare function

    • PKCS11 module

    • Access Manager exit

  • Fixes for static analysis findings.

  • OpenSSL Provider updated to version 1.1.2 (Provider code is also uploaded at - https://github.com/NXPPlugNTrust/se05x-openssl-provider)
    • 1.1.2 updates

      • ECDSA functions handle different formats of SHA algorithm string (Example - “sha256” / “SHA256” / “SHA2-256”)

    • 1.1.1 updates

      • Bug fix: Signature length calculation fixed in ECDSA functions.

    • 1.1.0 updates

      • CSR generation extended for all SHA algorithms now. (Get context functions - ‘sss_rsa_signature_get_ctx_params’ and ‘sss_ecdsa_signature_get_ctx_params’ updated to handle all SHA algorithms).

      • TLS use case : Ephemeral key is generated on host by default. (Enable ‘SSS_ENABLE_SE05X_EC_KEY_GEN_WITH_NO_KEYID’ in sssProvider_key_mgmt_ec.c file to generate on secure element).

      • Performance improvement : Provider is updated to store the client / server public key on host to avoid multiple secure element reads during TLS connection.

      • Bug fix - Correct RSA encrypt algorithm is set in function sss_rsa_enc_set_ctx_params, when ‘OSSL_ASYM_CIPHER_PARAM_PAD_MODE’ parameter data is integer type.

      • OSSL algorithms are updated with algorithm properties.

      • Enable / Disable random number generation in nxp provider using compile time option - ‘SSS_PROV_DISABLE_SE05X_RNG’. (Disabled by default)

      • Key id for RSA key generation can be passed to provider via ‘OSSL_PKEY_PARAM_RSA_FACTOR2’ parameter.

      • ECC / RSA key management import functions added - to handle reference keys. If the input key is not reference key, the function returns error to roll back on other available providers.

      • ECC key management - match and duplicate functions added.

      • ECDSA digest verify support added (function - sss_ecdsa_signature_digest_verify).

      • RSA digest verify support added (function - sss_rsa_signature_digest_verify).

      • RSA generate Key feature will generate RSA-CRT type keys by default. (Use compile time define SSS_PROV_GENERATE_RSA_PLAIN to change the RSA key type to plain.)

    • 1.0.3

      • ECC and RSA TLS use case extended for reference keys in file format also.

  • Linux TLS Client Server examples, AWS and Azure cloud examples are updated to work with OpenSSL Provider also (in addition to existing OpenSSL engine support)

  • PKCS11 updated to version to 4.8 (PKCS11 code is also uploaded at - https://github.com/NXPPlugNTrust/se05x-pkcs11)
    • 4.8 Updates

      • Global TLS context initialization is removed as cloud example doesn’t have dependency on TLS context

      • PKCS11 session handling clean up to support only single session

      • HKDF support added for PKCS11 specification version greater than or equal to 3.x

      • RSA_CRT support added for RSA key generation, sign/verify. (compile with PKCS11_ENABLE_RSA_KEY_GEN_CRT cmake option for CRT. By default plain RSA is enabled)

    • 4.7 Updates

      • Breaking: KeyID Handling: CKA_ID attribute (–id) passed is treated as byte array to avoid byte swapping. ID can be passed without changing the endianness now.

      • EC curve support extended for NIST-K and Brainpool.

      • PBKDF2 support added.

      • Bug fix: ASN-1 conversion in C_Verify function. PKCS11 implementation can now handle both raw and ASN-1 signature format for ECDSA verify.

2.4.5. SEMS Lite Change¶

  • Comparison of public key from secure element and SEMS Lite script (PkSemsCaAut field) is done to determine whether the applet can be updated.

2.4.6. File/Folder relocation¶

  • se05x_apis.h is renamed to se05x_reset_apis.h (file path - simw-top/hostlib/hostLib/platform/inc).

2.4.7. Build system changes¶

  • New cmake option PTMW_SE_RESET_LOGIC is added to select the reset logic for SE051/SE052.

  • frdmmcxn947 and frdmmcxa153 options are added for PTMW_Host cmake option.

  • New cmake options for EdgeLock 2GO hostname and port selection (EDGELOCK2GO_HOSTNAME and EDGELOCK2GO_PORT)

2.4.8. APIs & enum/types Changes¶

  • se05x_ic_reset API updated to receive applet version as input parameter. This is used to decide the reset logic for SE051 and SE052. (file path - simw-top/hostlib/hostLib/platform/inc/se05x_reset_apis.h)

2.4.9. EdgeLock 2GO Agent Changes¶

  • EdgeLock 2GO Agent enablement on top of OpenSSL 3.x using built-in sssProvider

  • EdgeLock 2GO hostname and port selection via command line or environment variable (only Windows/Linux targets) or preprocessor define.
